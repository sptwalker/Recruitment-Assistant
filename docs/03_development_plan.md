# 文档 3：分阶段开发测试计划

## 阶段 P0：需求确认与合规边界

### 目标

明确项目边界、优先平台、数据库、数据安全策略、登录方式与采集频率。

### 输出

- 首期支持平台已确认为 `智联招聘`。
- 数据库已确认为 `PostgreSQL`。
- 已确认保存联系方式明文，同时保存哈希和脱敏值。
- 已确认为 Windows 本地本人单用户使用，首期不做用户权限系统。
- 已确认保存简历附件原文件。

### 验收标准

- 技术选型冻结。
- 数据库结构冻结第一版。
- 首个平台采集流程明确。

## 阶段 P1：项目骨架与基础设施

### 开发内容

- Python 项目结构初始化。
- 配置管理。
- 日志系统。
- 数据库连接。
- Alembic 迁移框架。
- Streamlit 基础页面。

### 测试内容

- 启动 Streamlit 面板。
- 连接数据库成功。
- 日志正常输出。
- 配置可从 `.env` 加载。

### 验收标准

- 本地一条命令启动 Web 面板。
- 数据库迁移可执行。

## 阶段 P2：核心数据模型与 CRUD

### 开发内容

- SQLAlchemy ORM 模型。
- Pydantic Schema。
- 候选人、简历、岗位基础 CRUD。
- Excel 导出雏形。

### 测试内容

- 候选人创建、查询、更新、软删除。
- 简历结构化数据入库。
- 岗位创建与查询。
- Excel 导出字段正确。

### 验收标准

- 可手动录入候选人和岗位。
- 可导出候选人 Excel。

## 阶段 P3：单平台登录与登录态维护

### 首期平台

首期优先测试并开发 `智联招聘`，采用人工扫码/短信登录后保存 Playwright 登录态。其他平台在智联招聘闭环稳定后再扩展。

### 开发内容

- Playwright 浏览器管理。
- 人工登录入口。
- 登录态保存与恢复。
- 登录态有效性检测。

### 测试内容

- 首次人工登录。
- 重启程序后恢复登录态。
- 登录失效后提示重新登录。

### 验收标准

- 不保存平台账号明文密码。
- 可以稳定进入智联招聘目标简历页面。

## 阶段 P4：单平台简历采集闭环

### 开发内容

- 简历列表读取。
- 简历详情页访问。
- 原始 HTML / JSON 快照保存。
- 简历附件原文件下载并保存到 `data/attachments`。
- 原始数据入库 `raw_resume`。
- 附件记录入库 `resume_attachment`。
- 解析为标准 `candidate` / `resume` 数据。

### 测试内容

- 采集 5 条、20 条、50 条简历。
- 断点失败重试。
- 页面结构变化时保留快照用于调试。

### 验收标准

- 单平台从登录到入库全流程跑通。
- 简历核心字段准确率达到可接受水平。

## 阶段 P5：清洗、去重、字段统一

### 开发内容

- 手机、邮箱、姓名、城市、薪资、学历标准化。
- 候选人去重服务。
- 平台字段映射规则。
- 标签初版。

### 测试内容

- 重复候选人合并。
- 不同平台字段映射一致。
- 薪资、学历、工作年限格式统一。

### 验收标准

- 重复简历不会大量产生重复候选人。
- Excel 导出字段稳定统一。

## 阶段 P6：Streamlit 管理面板完善

### 开发内容

- 任务看板。
- 候选人列表与详情。
- 岗位管理。
- 导出中心。
- 简单筛选：关键词、城市、岗位、学历、经验、平台。

### 测试内容

- 页面操作完整闭环。
- 大量候选人分页/筛选流畅。
- 导出结果符合筛选条件。

### 验收标准

- 非技术用户可通过面板完成常规操作。

## 阶段 P7：多平台适配

### 开发内容

- 抽象平台适配器。
- 逐个平台增加：智联、前程无忧、猎聘、BOSS。
- 每个平台独立配置采集频率。

### 测试内容

- 各平台登录态恢复。
- 各平台字段映射测试。
- 多平台候选人去重。

### 验收标准

- 至少两个平台稳定采集并统一入库。

## 阶段 P8：岗位匹配评分与评价

### 开发内容

- 岗位 JD 管理。
- 规则评分：技能、经验、学历、城市、薪资。
- 简历推荐排序。
- 自动标签。

### 测试内容

- 同一岗位下候选人评分排序。
- 缺失项、匹配项说明准确。
- 分数可解释。

### 验收标准

- 可辅助筛选简历，而不是只做数据收集。

## 阶段 P9：稳定性与交付

### 开发内容

- 错误处理。
- 重试机制。
- 导出模板优化。
- 本地部署脚本。
- 操作说明。

### 测试内容

- 长时间运行测试。
- 异常网络测试。
- 登录失效测试。
- 数据库备份恢复测试。

### 验收标准

- Windows 本地可稳定运行。
- 基础问题可通过日志定位。

## 当前建议的 MVP 范围

第一版建议只做：

1. PostgreSQL。
2. 智联招聘人工登录态维护。
3. 简历列表 + 详情采集。
4. 简历附件原文件保存。
5. 原始数据保存。
6. 标准化入库。
7. 候选人去重。
8. 通用 Excel 导出。
9. Streamlit 基础面板。
10. 岗位 JD 录入与基础规则评分。

暂缓：

- 四个平台同时开发。
- 自动绕过验证码。
- 代理 IP 池。
- 大模型评分。
- Word 复杂模板导出。

## 已确认的首期执行细节

1. 智联招聘采集入口：`主动搜索候选人` 与 `已投递简历`。
2. 通用 Excel 模板：默认包含手机号、邮箱明文。
3. 简历附件命名规则：`平台_候选人姓名_手机号后四位_日期_原文件名`。
4. 岗位 JD 格式：`Word`，首期支持上传/读取 Word 文档并提取文本。
5. 智联招聘采集频率：采用保守策略，每次最多 `20-50` 份，每份间隔 `8-30` 秒。

## 开工状态

需求确认已完成，可以进入 `P1：项目骨架与基础设施` 开发。
