## 二、功能变更明细

### 2.1 BOSS 直聘采集精确 DOM 定位

**问题**：旧版使用 text-search 启发式方法定位 DOM 元素，存在误判导致"候选人列表已逛完"假阳性。

**改进**：通过 Chrome DevTools Protocol（CDP）逐一验证所有 DOM 选择器，替换为精确 CSS 选择器，每个均保留旧方法作为 fallback。

| 功能 | 旧选择器 | 新选择器 |
|------|---------|---------|
| 沟通菜单 | text-search 遍历 | `dl.menu-chat a[href*="/web/chat"]` |
| 沟通中 Tab | text-search | `.chat-label-item[title="沟通中"]` |
| 候选人列表容器 | 多层启发式 | `div.user-list.b-scroll-stable` |
| 候选人条目 | 通用子元素 | `.geek-item-wrap > .geek-item` |
| 简历按钮 | text-search | `a.btn.resume-btn-file` |
| 简历预览 iframe | 遍历 iframe | `iframe.attachment-iframe` |
| 关闭预览按钮 | text-search | `.dialog-wrap.active .close-btn` |
| 下载按钮 | 遍历 popover | `.attachment-resume-btns .popover.icon-content:last-child` |

**滚动机制重写**：利用 Vue 虚拟列表组件 `boss-virtual-list` 的 API：
- `scrollToOffset()` — 主滚动方法
- `scrollToBottom()` / `scrollToIndex()` — 触底/精确定位 fallback
- 使用 `processedCount` 变量 + WeakSet 双重计数避免遗漏

**测试结果**：连续 11/11 候选人采集成功，无假阳性终止。

### 2.2 简历管理页面 UI 优化

- **信息卡片重构**：5 列单行布局（简历库总数 + 待入库文件 + 3 个平台卡片）
- **SVG 环形百分比图**：BOSS（绿 #0a7d2e）、智联（蓝 #1d4ed8）、51（橙 #b45309）
- **字体层级**：简历库总数黑体 16px/900，平台卡片 15px/700
- **按钮对齐**：与日期选择器底部对齐，统一间距
- **待入库文件指标**：扫描结束后自动更新

### 2.3 面试管理 — AI 面试大纲

#### 新增数据表：`interview_outlines`

| 字段 | 类型 | 说明 |
|------|------|------|
| outline_id | INTEGER PK | 自增主键 |
| invitation_id | INTEGER UNIQUE FK | 关联面试邀约（一对一） |
| candidate_id | INTEGER FK | 关联候选人 |
| position_id | INTEGER FK | 关联岗位（可空） |
| content | TEXT | Markdown 大纲正文 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### AI 提示词升级

从简单的"生成 5-8 个面试问题"升级为专业结构化面试大纲提示词：
- 角色：资深企业招聘面试官 & 人才测评专家
- 方法论：胜任力模型 + STAR 行为面试法
- 输出格式：6 模块强制结构（破冰→专业→经历深挖→软素质→动机→反问）
- 评分权重：100 分制 5 维度
- 层级适配：基层重执行、骨干重专业、管理重统筹

候选人摘要增加：教育经历、项目经验（描述/职责/成果）、求职意向。

#### UI 交互

- `@st.dialog("面试大纲")` 弹窗展示
- 自动缓存：首次生成后存入 DB，再次打开直接读取
- "重新生成"按钮：传入最新岗位需求 + 历史面试评价
- "打印大纲"按钮：Word 标准排版打印（A4/宋体 12pt/黑体标题/1.8 行距）
